# MedCore HMS Documentation Hub

This documentation set is organized for product, engineering, operations, and compliance teams working across multiple regions and hospital tenants.

## Quick Navigation

### Core Product Docs
- `blueprint.md` - master product and platform blueprint.
- `TASKLIST.md` - implementation and delivery tracker.

### Dashboard Docs
- `dashboard/README.md` - dashboard docs index.
- `dashboard/ARCHITECTURE.md` - role-aware dashboard architecture.
- `dashboard/DESIGN_TASKS.md` - dashboard UX and implementation tasks.

### Architecture
- `architecture/README.md` - system-level architecture decisions and boundaries.

### Development
- `development/SETUP.md` - local setup and onboarding.
- `development/CONTRIBUTING.md` - team contribution workflow.

### API and Security
- `api/API_STANDARDS.md` - API design conventions and response format.
- `security/SECURITY_MODEL.md` - authz/authn model and security controls.

### Product Localization and Branding
- `product/INTERNATIONALIZATION.md` - i18n, l10n, timezone, and locale strategy.
- `product/BRAND_GUIDELINES.md` - global brand system and regional adaptation.

### Operations, Testing, and Release
- `operations/RUNBOOK.md` - production operational runbook.
- `testing/TEST_STRATEGY.md` - test layers and quality gates.
- `release/RELEASE_PROCESS.md` - release lifecycle and rollout model.

## Documentation Standards

- Keep docs actionable and versioned with code changes.
- Include owner, last updated date, and scope for critical documents.
- Prefer short sections, checklists, and explicit examples.
